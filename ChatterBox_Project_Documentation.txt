PROJECT TITLE: ChatterBox - Real-Time WebSocket Chat Application

------------------------------------------------------------------------

1. PROJECT OVERVIEW

ChatterBox is a real-time web-based chat application developed using
FastAPI and WebSockets. The system allows multiple users to communicate
instantly through a browser interface. The application includes user
authentication, real-time messaging, message status tracking, emoji
support, toxic message detection, and user blocking functionality.

------------------------------------------------------------------------

2. TECHNOLOGIES USED

Backend: - FastAPI (Python Web Framework) - WebSockets (Real-time
communication) - SQLite (Database)

Frontend: - HTML - CSS - JavaScript - WebSocket API (Browser side)

------------------------------------------------------------------------

3. PROJECT STRUCTURE

1.  main.py
    -   Core backend file.
    -   Handles routing, WebSocket connections, message logic, and
        database operations.
2.  register.html
    -   User registration page.
    -   Collects username and password and stores in database.
3.  login.html
    -   User login page.
    -   Authenticates user credentials.
4.  chat.html
    -   Main chat interface.
    -   Handles:
        -   Sending messages
        -   Receiving messages
        -   Emoji selection
        -   Message status (Sent, Delivered, Read)
        -   Date display (Today / Yesterday)
        -   Logout functionality
5.  database.db
    -   SQLite database.
    -   Stores:
        -   Users table
        -   Messages table
        -   Message status
        -   Warning counts
        -   Blocked users

------------------------------------------------------------------------

4. DATABASE DESIGN

USERS TABLE: - username (Primary Key) - password - warnings (default
0) - blocked (default 0)

MESSAGES TABLE: - id (Primary Key) - sender - message - timestamp -
status (sent, delivered, read)

------------------------------------------------------------------------

5. SYSTEM FEATURES

1.  User Authentication
    -   Users must register before logging in.
    -   Login validates credentials.
    -   Blocked users cannot log in.
2.  Real-Time Messaging
    -   Implemented using WebSockets.
    -   Messages are instantly broadcast to connected users.
3.  Message Status System
    -   ✓ : Message Sent
    -   ✓✓ (Grey): Message Delivered
    -   ✓✓ (Blue): Message Read
    -   Read status is triggered when the receiver becomes active in the
        chat.
4.  Emoji Support
    -   Emoji picker in chat interface.
    -   Selected emoji is appended to input field.
5.  Toxic Message Detection
    -   System checks for predefined stop words.
    -   First offense: Warning 1
    -   Second offense: Warning 2
    -   Third offense: User blocked automatically
6.  Date Separator
    -   Messages are grouped by date.
    -   Displays “Today” and “Yesterday” labels when applicable.
7.  Logout Feature
    -   Clears user session.
    -   Redirects to login page.

------------------------------------------------------------------------

6. WORKFLOW EXPLANATION

Step 1: User registers and login credentials are stored in database.

Step 2: After login, user connects to WebSocket endpoint.

Step 3: When a user sends a message: - Message is saved in database with
status “sent”. - Message is broadcast to all active users.

Step 4: When receiver gets the message: - Delivered status is updated. -
Sender sees grey double tick.

Step 5: When receiver becomes active: - Read status is updated. - Sender
sees blue double tick.

------------------------------------------------------------------------

7. SECURITY & MODERATION

-   Toxic word filtering prevents misuse.
-   Automatic blocking after 3 violations.
-   Session validation prevents unauthorized access.

------------------------------------------------------------------------

8. CONCLUSION

ChatterBox demonstrates implementation of real-time communication using
WebSockets. The system successfully integrates authentication,
moderation, status tracking, and an interactive frontend interface.

This project showcases understanding of: - Backend API development -
WebSocket real-time systems - Database management - Frontend interaction
handling - User moderation mechanisms
